厂家端活动创建页面设计文档
一、页面整体结构
导航栏
活动基础信息区
合规声明区
服务模式选择区
病例要求区
医生筛选条件区
预览与提交区
二、详细设计
1. 导航栏
元素：返回按钮、保存按钮
交互：点击保存按钮后，弹出确认弹窗 “是否确认保存当前内容？”，有 “确认” 和 “取消” 两个选项。
注意事项：保存按钮在未填写必填字段时置灰不可点击。
2. 活动基础信息区
字段 1：活动名称
类型：单行输入框
规则：必填，最多输入 50 个字符
提示：“请输入活动名称，如‘儿童上呼吸道感染病例征集活动’”
字段 2：活动主题
类型：单行输入框
规则：必填
交互：输入内容后，下方自动弹出细化选项下拉菜单，包含 “年龄段”“症状表现”“治疗阶段” 等选项，可多选
提示：“请输入核心主题，如‘儿童上呼吸道感染’，系统将自动推荐相关细化选项”
字段 3：活动背景
类型：多行输入框
规则：选填，最多输入 200 个字符
提示：“可简要说明活动目的，如‘为配合学术会议，征集高质量教学病例’”
字段 4：活动时间范围
类型：日期选择器（开始日期、结束日期）
规则：必填，结束日期需晚于开始日期
提示：“请选择活动起止时间，建议征集周期不少于 7 天”
字段 5：目标采购数量
类型：数字输入框
规则：必填，最小值为 1，最大值为 1000
提示：“请输入计划采购的病例数量”
字段 6：活动要求
类型：多行输入框
规则：选填，最多输入 300 个字符
提示：“可补充其他要求，如‘需包含用药前后症状对比’”
3. 合规声明区
元素：合规声明文本、“我已阅读并同意” 勾选框
声明文本：“1. 本次活动仅可征集有教学意义的互动病例片段，不可收集完整患者病历及隐私信息；2. 不可指定具体医生参与活动；3. 平台仅对病例进行合规合法性审核，不承担学术质量责任（选择高级服务除外）。”
规则：必填，需勾选后才能进行下一步操作
样式：声明文本用红色字体标注，勾选框位于文本左侧
提示：“请仔细阅读并勾选，否则无法创建活动”
4. 服务模式选择区
引导语：“请选择活动服务模式，不同模式将为您提供不同的病例质量保障”
卡片 1：基础服务
标题：基础服务（黑色加粗）
适用场景：“适合需要快速征集合规教学病例的活动”（灰色小字）
功能点：
合规合法性双重校验（系统自动校验 + 人工二次核查，确保病例不涉及完整患者信息、不违反医疗数据规范）
病例格式标准化处理（自动统一字体、排版，生成可直接分发的教学文档，省去厂家格式调整成本）
学习数据实时追踪（提供医生学习人数、学习时长、答题正确率等核心数据报表，支持导出 Excel）
标签：绿色 “推荐” 标签，标注 “80% 新用户首次创建活动选择此模式”
卡片 2：高级服务（专家评审）
标题：高级服务（专家评审）（蓝色加粗）
适用场景：“含基础服务全部功能，额外提供学术质量保障”（橙色小字）
额外功能点：
三甲医院相关科室专家评审（可指定专家领域，如 “儿科呼吸科”，专家会从教学逻辑、医学准确性、案例典型性三个维度打分，给出修改建议）
病例优化指导（专家针对不足的病例，提供具体修改方向，比如 “建议补充儿童上呼吸道感染不同年龄段用药剂量对比”，帮助医生提升病例质量）
专家点评附在病例末尾（分发时自动带上专家姓名、职称和点评内容，增加病例权威性，方便学习医生理解重点）
标签：橙色 “增值服务” 标签，标注 “适合学术会议、药企内部培训等高质量教学场景”
交互：
两种模式用单选按钮选择，默认选中基础服务
选择高级服务后，弹出小弹窗：“已为您开启专家评审服务，创建活动后可在‘专家对接’模块选择具体专家领域（含全国 300 + 三甲医院内科、儿科等科室专家）”，弹窗有 “知道了” 按钮
选择基础服务后，提示：“已为您开启合规保障服务，活动期间可随时查看合规校验报告”
注意事项：
基础服务的合规校验结果会在医生投稿后 1 个工作日内反馈，高级服务的专家评审需 3-5 个工作日
若选择高级服务，建议在 “活动要求” 中注明 “需配合专家评审修改病例”
两种服务均支持活动创建后升级，可在活动管理页操作
5. 病例要求区
字段 1：题目类型要求
类型：单选按钮组
选项：“仅单选题”“仅多选题”“两者均可”
规则：必填
附加选项：“是否需要题目解析”（复选框，选填）
提示：“请选择病例中互动题目的类型”
字段 2：题目数量范围
类型：滑块 + 数字输入框（两者联动）
规则：必填，范围 3-5 道
交互：拖动滑块可调整数量，也可直接在输入框输入数字
提示：“每篇病例需包含 3-5 道互动题目，以保证教学深度”
字段 3：病例主题细化选项
类型：下拉菜单（根据活动主题自动生成）
规则：选填，可多选
示例：若活动主题为 “儿童上呼吸道感染”，下拉菜单包含 “年龄段：1-3 岁 / 4-6 岁 / 7-12 岁”“症状表现：发热 / 咳嗽 / 鼻塞”“治疗阶段：急性期 / 恢复期” 等选项
提示：“选择细化选项，可帮助医生更精准地创作病例”
6. 医生筛选条件区
字段 1：医生职称
类型：下拉菜单
选项：“不限”“住院医师”“主治医师”“副主任医师及以上”
规则：选填
字段 2：医生科室
类型：下拉菜单（可搜索）
选项：“不限”“儿科”“呼吸科”“内科” 等所有临床科室
规则：选填
字段 3：医生从业年限
类型：下拉菜单
选项：“不限”“1-3 年”“4-6 年”“7 年及以上”
规则：选填
提示：“设置筛选条件，可精准定位符合要求的医生，无需指定具体个人”
注意事项：所有条件均为筛选参考，不保证完全匹配，平台会根据条件推荐符合要求的医生
7. 预览与提交区
元素：预览按钮、提交按钮
交互：
点击预览按钮后，打开新标签页，显示活动创建后的完整效果（包括活动名称、主题、要求、服务模式等所有信息），预览页有 “返回编辑” 按钮
点击提交按钮前，系统自动校验必填字段是否填写完整、格式是否正确
若校验通过，弹出 “活动创建成功！” 弹窗，包含 “查看活动”“继续创建” 两个选项
若校验不通过，在未填写 / 格式错误的字段下方显示红色提示文字（如 “请输入活动名称”“结束日期需晚于开始日期”）
注意事项：提交后活动将立即上线，如需修改需进入活动管理页操作
三、页面排版与样式
整体布局：从上到下按导航栏→活动基础信息区→合规声明区→服务模式选择区→病例要求区→医生筛选条件区→预览与提交区依次排列，各区域之间用灰色分隔线隔开
间距：各字段之间上下间距为 15px，左右间距为 20px；按钮之间间距为 10px
字体：标题用 16px 黑体，字段标签用 14px 宋体，提示文字用 12px 灰色宋体
颜色：必填字段标签前加红色 “*”，按钮颜色：保存按钮为蓝色，提交按钮为绿色，预览按钮为灰色，返回按钮为白色（带灰色边框）
四、交互优先级
字段填写时，实时显示格式错误提示（如输入非数字到目标采购数量框，立即显示 “请输入数字”）
点击保存后，自动保存当前已填内容，再次进入页面可继续编辑
预览页与编辑页内容实时同步，编辑页修改后，重新预览可看到最新内容
